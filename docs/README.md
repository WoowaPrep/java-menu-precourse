# 점심 메뉴 추천

---

## 기능 요구사항

### 입력 기능
- [X] 메뉴 추천을 받을 코치의 이름 입력 (, 로 구분해서 입력이 들어옴)
- [X] 각 코치가 못 먹는 메뉴를 입력 (, 로 구분해서 입력이 들어옴)
- [X] 각 과정을 코치 수 만큼 반복 입력


### 출력 기능
- [X] 서비스 시작 문구 출력
- [X] 서비스 종료 문구 출력
  - 메뉴 추천 결과입니다.
  - [ 구분 | 월요일 | 화요일 | 수요일 | 목요일 | 금요일 ]
  - [ 카테고리 | 한식 | 한식 | 일식 | 중식 | 아시안 ]
  - [ 토미 | 쌈밥 | 김치찌개 | 미소시루 | 짜장면 | 팟타이 ]
  - [ 제임스 | 된장찌개 | 비빔밥 | 가츠동 | 토마토 달걀볶음 | 파인애플 볶음밥 ]
  - [ 포코 | 된장찌개 | 불고기 | 하이라이스 | 탕수육 | 나시고렝 ]
  - 추천을 완료했습니다.


### 기능 요구사항
- [X] 코치들은 월, 화, 수, 목, 금요일에 점심 식사를 한다.
- [X] 메뉴를 추천하는 과정
  - 월요일에 추천할 카테고리를 무작위로 정한다.
  - 각 코치가 월요일에 먹을 메뉴를 추천한다.
  - 화, 수, 목, 금요일에 대해 i, ii 과정을 반복한다.
  

### 예외 처리
- [X] 코치의 이름은 2글자 미만 또는 4글자 초과된 경우
- [X] 점심 식사 하는 코치의 수가 2명 미만, 5명 초과된 경우 
- [X] 각 코치의 못먹은 메뉴의 수가 2개 초과된 경우
  - 먹지 못하는 메뉴가 없으면 빈 값을 입력됨.
  - 추천을 못하는 경우는 발생하지 않으니 고려하지 않아도 됨. 
- [X] 한 주에 같은 카테고리가 2회 초과된 경우
  - 예시) 구구: 비빔밥, 김치찌개, 쌈밥, 규동, 우동 → 한식을 3회 먹으므로 불가능 
- [X] 각 코치에게 한 주에 중복된 메뉴를 추천한 경우
  - 예시) 토미: 비빔밥, 비빔밥, 규동, 우동, 볶음면 → 한 코치가 같은 메뉴를 먹으므로 불가능 
- [X] 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException를 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받음.
